If your website doesn't look right on mobile devices, try these solutions.

## Common Mobile Issues

### Content Overflowing

**Symptoms:**
- Horizontal scrolling on mobile
- Content extends beyond screen

**Solutions:**

1. **Check image sizes**
   - Use responsive images
   - Set max-width: 100%

2. **Review text content**
   - Long words may overflow
   - Add word-break for long URLs

3. **Use AI to fix**
   ```
   Fix the mobile layout - content is overflowing horizontally
   ```

### Text Too Small

**Symptoms:**
- Text hard to read on mobile
- Users need to zoom

**Solutions:**

1. **Check font sizes**
   - Minimum 16px for body text
   - Headings should scale appropriately

2. **Use AI to adjust**
   ```
   Increase text size for mobile readability
   ```

### Buttons Too Small

**Symptoms:**
- Hard to tap buttons
- Accidental clicks

**Solutions:**

1. **Increase tap targets**
   - Minimum 44x44px for buttons
   - Add padding around links

2. **Use AI to fix**
   ```
   Make buttons larger and easier to tap on mobile
   ```

## Testing Mobile Layout

### In Lindo.ai Editor

1. Click mobile preview icon
2. Review all sections
3. Test interactions

### On Real Device

1. Publish the page
2. Open on your phone
3. Test all functionality

### Browser DevTools

1. Open DevTools (F12)
2. Toggle device toolbar
3. Select mobile device

## Fixing Layout Issues

### Using AI Agents

Ask the AI to fix specific issues:

```
Fix the hero section - it's not displaying correctly on mobile
```

```
Make the navigation menu mobile-friendly
```

```
Adjust the grid layout for mobile screens
```

### Common Fixes

| Issue | AI Prompt |
|-------|-----------|
| Overlapping elements | "Fix overlapping elements on mobile" |
| Hidden content | "Ensure all content is visible on mobile" |
| Broken navigation | "Fix the mobile navigation menu" |
| Image sizing | "Optimize images for mobile display" |

## Prevention Tips

### Design Mobile-First

- Start with mobile layout
- Add complexity for larger screens
- Test frequently during design

### Use Responsive Patterns

- Flexible grids
- Scalable images
- Readable typography

### Test Regularly

- Preview on multiple devices
- Check after each major change
- Test on real devices when possible

## When to Rebuild

If issues are extensive:

1. Consider regenerating the section
2. Use mobile-focused prompt
3. Specify mobile requirements upfront
