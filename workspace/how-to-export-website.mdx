title: "How to Export a Website"
description: "Downloading website as ZIP file"
---

Download your website as a ZIP file for backup or self-hosting.

<Steps>
  <Step title="Find the Website">
    Locate the website in your Websites Grid.
  </Step>
  <Step title="Open Actions Menu">
    Hover over the website card to reveal actions.
  </Step>
  <Step title="Click Export">
    Click **Export** from the actions menu.
  </Step>
  <Step title="Download ZIP">
    Your browser downloads a ZIP file containing your website.
  </Step>
</Steps>
## What's Included

The ZIP file contains:
- **HTML files** - All pages
- **CSS** - Tailwind styles
- **Images** - All uploaded images
- **Assets** - Fonts, icons, etc.

## What's Not Included

- Analytics data
- CRM/customer data
- Server-side functionality
- Lindo.ai hosting

## ZIP Structure

```
website-export/
├── index.html
├── about.html
├── contact.html
├── css/
│   └── styles.css
├── images/
│   ├── hero.jpg
│   └── logo.png
└── assets/
    └── ...
```

## Use Cases

### Backup

Keep a local copy of your website:
1. Export periodically
2. Store in cloud storage
3. Have a backup if needed

### Self-Hosting

Host on your own server:
1. Export the website
2. Upload to your hosting provider
3. Configure your domain

### Handoff to Developer

Share with a developer:
1. Export the website
2. Send the ZIP file
3. Developer can modify code directly

### Archive Projects

Archive completed projects:
1. Export when project ends
2. Store for records
3. Free up active website slots

## Hosting Exported Sites

To host an exported site:

1. **Static hosting** - Netlify, Vercel, GitHub Pages
2. **Traditional hosting** - Upload via FTP
3. **CDN** - CloudFront, Cloudflare Pages

## Limitations

Exported sites are static HTML:
- No AI editing capability
- No built-in analytics
- No CRM functionality
- Manual updates required

## Tips

- **Export before deleting** - Keep a backup
- **Test locally** - Open index.html to verify
- **Update links** - May need adjustment for new host
