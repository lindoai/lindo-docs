Different roles have different permissions in your workspace.

## Role Hierarchy

From most to least access:
1. Owner
2. Admin
3. Editor
4. Viewer

## Permission Matrix

| Permission | Owner | Admin | Editor | Viewer |
|------------|-------|-------|--------|--------|
| View websites | ✓ | ✓ | ✓ | ✓ |
| Edit websites | ✓ | ✓ | ✓ | - |
| Create websites | ✓ | ✓ | ✓ | - |
| Delete websites | ✓ | ✓ | - | - |
| Manage team | ✓ | ✓ | - | - |
| Access settings | ✓ | ✓ | - | - |
| Billing access | ✓ | - | - | - |
| Delete workspace | ✓ | - | - | - |

## Role Details

### Owner

The workspace creator:
- Full control
- Billing management
- Cannot be removed
- Only one per workspace

### Admin

Trusted team members:
- Manage other members
- Access all settings
- Cannot access billing
- Cannot delete workspace

### Editor

Content creators:
- Create and edit websites
- Use AI features
- No admin access
- No settings access

### Viewer

Read-only access:
- View websites
- Cannot edit
- Cannot create
- Observation only

## Choosing Roles

| Use Case | Recommended Role |
|----------|------------------|
| Business partner | Admin |
| Designer/developer | Editor |
| Client preview | Viewer |
| Contractor | Editor |

## Tips

- Use least privilege principle
- Review roles regularly
- Limit admin access
- Use viewer for clients
