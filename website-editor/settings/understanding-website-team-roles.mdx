---
title: "Understanding Website Team Roles"
description: "Permission levels for website collaborators"
---

Different roles have different permissions when collaborating on a website.

## Available Roles

### Editor
Can:
- Edit pages and content
- Create new pages
- Manage products
- View analytics

Cannot:
- Delete the website
- Manage team members
- Change critical settings

### Viewer
Can:
- View pages and content
- Access preview
- View analytics

Cannot:
- Make any changes
- Edit content
- Manage products

## Role Assignment

When inviting team members:
1. Enter their email
2. Select their role
3. Send invitation

## Changing Roles

To change a team member's role:
1. Remove their current access
2. Re-invite with the new role

## Best Practices

- Give minimum necessary permissions
- Use Viewer for stakeholders who only need to review
- Use Editor for active collaborators
- Review team access periodically
